# KinetIX Deployment Guide — Railway Free Tier

## ⚠️ Important: Test Data Only
**This deployment uses a free-tier PostgreSQL database. Do NOT store real financial data.**
- Use mock transactions for testing only
- Data may not be encrypted at rest
- No backup guarantee on free tier
- For production, migrate to Railway paid tier ($7+/month)

---

## Quick Start (5 minutes)

### Step 1: Create Railway Account
1. Go to [railway.app](https://railway.app)
2. Sign up with GitHub (recommended — auto-connects to your repo)

### Step 2: Create New Project
1. Click "Create New Project"
2. Select "Deploy from GitHub repo"
3. Authorize Railway to access your GitHub
4. Select `kineticrl76/kinetix` repository

### Step 3: Configure Environment Variables
Railway will prompt you to add env vars. Add these:

```
DATABASE_URL=postgresql://[auto-generated by Railway]
DIRECT_URL=postgresql://[auto-generated by Railway]
NEXT_PUBLIC_BASE_URL=https://[your-railway-domain].railway.app
GOOGLE_CLIENT_ID=[your Google OAuth client ID]
GOOGLE_CLIENT_SECRET=[your Google OAuth client secret]
VISION_API_URL=https://api.groq.com/openai/v1
VISION_API_KEY=[your Groq API key]
SMS_WEBHOOK_TOKEN=[generate random secret]
UPLOAD_DIR=./public/uploads/receipts
UPLOAD_BASE_URL=/uploads/receipts
```

**Note**: Railway auto-generates `DATABASE_URL` and `DIRECT_URL` when you add PostgreSQL. Copy those values.

### Step 4: Add PostgreSQL Database
1. In Railway project dashboard, click "+ Add service"
2. Select "PostgreSQL"
3. Railway auto-populates `DATABASE_URL` and `DIRECT_URL` env vars

### Step 5: Deploy
1. Click "Deploy" — Railway watches your GitHub repo
2. On first deploy, Prisma will auto-run migrations
3. Database tables created fresh (no test data)

### Step 6: Get Your App URL
Once deployed:
- Your app runs at: `https://[random-name].railway.app`
- Share this URL to test the app

## Troubleshooting

### Build fails
- Check "Deploy" tab in Railway for error logs
- Common: Missing env vars (see Step 3)
- Check that Prisma schema is valid: `npx prisma validate`

### App crashes
- Railway Logs tab shows errors
- Common: `DATABASE_URL` not set or invalid PostgreSQL connection

### Slow startup
- Free tier cold starts take 30-60 seconds (normal)
- Upgrade to paid for instant starts
